# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

`headroom-setup.sh` is a single-file bash installer that wires Claude Code usage tracking and push notifications into `~/.claude/`. Running it is the entire workflow — there is no build system, package manager, or test suite.
The scripts generated by this setup runs on both macos(zsh) and linux(bash).

## Running the script

```bash
bash headroom-setup.sh
```

The script is interactive: it prompts for an optional brrr.now secret to enable push notifications. To test non-interactively, pipe a blank line:

```bash
echo "" | bash headroom-setup.sh
```

To test idempotency (re-running on an already-configured system), just run it again — each step checks before writing.

## Architecture of the generated system

The installer embeds five scripts as heredocs and writes them to `~/.claude/headroom/`:

| File | Role |
|------|------|
| `statusline.sh` | Claude Code `statusLine` hook — receives JSON on every prompt, writes it to `headroom-usage.json`, optionally launches `notify.sh` |
| `headroom.sh` | Terminal display: reads `headroom-usage.json` and renders session/week/context bars with reset countdowns. Supports `--watch [N]` for auto-refresh. |
| `notify.sh` | Threshold notifier — runs in background after each prompt, sends brrr.now alerts at 80% and 90% usage. Uses `.notify-state` to debounce. |
| `hook-notify.sh` | Hook event notifier — wired into every Claude Code hook event type; converts the hook JSON to a titled push notification. |
| `config.json` | Stores `brrr_secret` (mode 600). |

**Data flow:** Claude Code → `statusline.sh` → `headroom-usage.json` → `headroom.sh` (display) / `notify.sh` (alerts)

**Settings patching:** The installer uses inline Python (`python3 -c`) to read/merge/write `~/.claude/settings.json` as JSON rather than regex-editing it, preserving existing keys.

**Hook event coverage:** `hook-notify.sh` handles: `SessionStart`, `SessionEnd`, `UserPromptSubmit`, `PreToolUse`, `PostToolUse`, `PostToolUseFailure`, `Stop`, `StopFailure`, `Notification`, `SubagentStart`, `SubagentStop`, `PreCompact`, `PostCompact`, `TaskCompleted`, `ConfigChange`.

## Key implementation patterns

- All file paths are built from `$HOME` so the script is user-portable.
- `notify.sh` uses a lock directory (`mkdir` atomic) to prevent concurrent runs.
- Log files are capped at 100 lines via `tail -n 99` before each append.
- The brrr.now secret is normalised at read time: both raw tokens (`br_usr_...`) and full URLs are accepted.
